- 任务跟踪：`docs/沉浸式角色扮演模块-视觉与长文富文本任务清单.md`

## 1. 背景与目标

当前沉浸式角色扮演模块已经具备基础剧情、阅读模式、AI 帮写、协议输出与长文模式能力，但还存在两类明显短板：

1. **视觉层级不稳定**
   - 主对话页仍然偏“聊天页换皮”，固定冷蓝强调色与背景图经常不匹配。
   - 长文阅读页氛围尚可，但连续阅读时可读性不足，文本和背景互相抢注意力。
2. **长文模式的文本辨识度不够**
   - 当前只对引号内容做了轻度高亮，无法稳定区分“角色本人说的话”和普通叙述。
   - 用户需要更清楚地区分角色台词、叙述、心理描写，从而提升剧情阅读效率。

本轮改造的核心目标不是单纯“换色”，而是建立一套同时满足 **沉浸互动** 与 **长文阅读** 的双模式体验：

- **对话页保持沉浸感**：强调角色现场感、背景氛围和轻玻璃质感。
- **长文页切换到阅读优先**：强调正文可读性、层级清晰和连续阅读舒适度。
- **富文本高亮只服务于角色本人台词**：不是高亮所有引号，而是更准确地突出角色输出。
- **配色跟随背景图动态取色**：减少“一律冷蓝化”，让不同场景有自己的气质。

## 2. 已确认的产品决策

### 2.1 页面策略

- 对话页：保持沉浸感，不强行做成阅读器。
- 长文页：允许单独使用更偏阅读器的 UI，前提是阅读体验明确提升。

### 2.2 富文本识别策略

用户已明确选择：**只高亮角色本人说的话**。

这意味着不能仅靠“所有引号都上色”来实现，因为自然文本里无法稳定推断发言者。为保证准确率，本轮采用“内部渲染标记”方案：

- 允许模型在长文模式中输出隐藏语义标签：
  - `<char>...</char>`：角色本人直接说出口的话
  - `<thought>...</thought>`：角色本人的心理活动
- 标签只供客户端渲染使用，**正式展示内容中绝不能露出标签本身**。

### 2.3 标签使用约束

- 标签只允许出现在长文模式。
- 标签不做嵌套。
- 标签只包裹最小语义片段。
- 其他人物对白、环境描写、普通叙述不加标签。
- 用户最终看到的正文、复制内容、阅读页内容、流式展示，都必须是无标签的正式文本。

## 3. 本轮设计原则

### 3.1 对话页：沉浸优先

目标：像“剧情现场互动”，而不是通用 IM 聊天窗口。

设计原则：

- 保留背景图与场景感。
- 顶部、输入区、建议区统一使用同一套沉浸式玻璃 token。
- 强调色不再固定为冷蓝，而由背景图动态派生。
- 角色消息、用户消息、旁白、系统提示具有明确语义层级。

### 3.2 长文页：阅读优先

目标：能连续阅读 5~10 屏仍保持舒适。

设计原则：

- 背景图负责氛围，阅读容器负责可读性。
- 正文优先左对齐，减少大段居中和过度样式化。
- 角色台词突出，但保持克制，不做大面积高饱和染色。
- 用户消息降级为“阅读中的玩家输入记录”，不与正文争主位。

### 3.3 富文本：排版为主，颜色为辅

长文模式的层级区分优先依赖：

- 段落结构
- 字重
- 明暗对比
- 少量语义色

而不是过度依赖鲜艳颜色。最终目标是“好辨认”而不是“花”。

## 4. 方案拆分

## Phase 1：基础能力与安全兜底

目标：先把“不会露标签”和“配色不串味”做对。

范围：

1. 长文模式提示词加入内部渲染标记说明。
2. 新增长文标记解析器，支持 `<char>` / `<thought>`。
3. UI Model 区分：
   - 展示用纯文本
   - 渲染用原始富文本源
4. 流式展示与复制链路确保不露标签。
5. 沉浸式动态配色从“强制偏冷蓝”改为“保留场景主色相 + 安全约束”。

## Phase 2：阅读模式重构

目标：把长文页从“展示层”升级为“阅读器层”。

范围：

1. 阅读模式增加稳定阅读容器。
2. 优化长文排版：字号、行高、段距、边距。
3. 用富文本准确高亮角色本人台词与心声。
4. 降低用户消息在长文回看中的视觉权重。

## Phase 3：对话页演出增强

目标：继续提升角色感和“现场感”。

范围：

1. 统一对话页角色强调色、用户强调色、旁白色、心声色。
2. 优化 emotion 展示与状态胶囊层级。
3. 视实现成本决定是否补 presence strip / portrait layer 的再次接入。

## 6. 风险与兜底

### 风险 1：标签在正式内容中泄漏

必须兜底的展示链路：

- 流式输出
- 对话页
- 阅读页
- 复制内容
- 摘要 / 记忆 / 剧情转录输入

兜底策略：

- 所有展示层优先使用剥离标签后的 `content` / `copyText`。
- 只有长文富文本渲染器使用 `richTextSource`。

### 风险 2：模型偶尔漏标或错误标注

兜底策略：

- 标签缺失时退回普通长文渲染，不强行误判所有引号。
- 标签异常时保留纯文本可读性，不因为解析失败导致整段不可见。

### 风险 3：动态取色过度偏暗或偏脏

兜底策略：

- 保留原始主色相，但对饱和度、明度做安全限制。
- 为正文、弱文本、边框、容器分别派生对比度安全色。

## 7. 验收标准

### 功能验收

- 长文模式支持 `<char>` / `<thought>` 内部渲染标签。
- 用户最终可见内容中不出现标签。
- 复制内容不出现标签。
- 长文回看不出现标签。
- 流式过程中不闪标签。
- 未打标签时仍能正常阅读。

### 视觉验收

- 不同背景图下强调色会跟随场景变化。
- 角色本人说的话一眼可辨。
- 长文连续阅读明显比旧版更舒适。
- 用户消息不会过度打断剧情阅读节奏。

2. 再同步提升阅读模式与动态配色。
