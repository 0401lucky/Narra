# 人工回归测试清单

> 目标：在当前重构与协议增强完成后，用最少但足够覆盖的手工验证，确认主要功能、关键联动与发布链路可用。

## 0. 执行规则

### 测试包
- 默认使用：`app/build/outputs/apk/debug/Narra-v1.0.6-dev-10006-dev.apk`

### 执行建议
- 优先顺序：P0 → P1 → P2
- 每项至少记录：
  - 结果：通过 / 失败
  - 失败现象
  - 截图或日志位置
  - 是否可稳定复现
- 若某项失败，先不要继续正式发包，先回到对应模块修复后回归。

### 推荐环境
- 至少一台常规 Android 设备 / 模拟器完成 P0
- Provider 协议相关测试尽量准备：
  - 一个 OpenAI Compatible 端点
  - 一个第三方自定义兼容端点
  - 一个支持 `/responses` 的第三方端点
  - 一个 Anthropic `/v1/messages` 端点
