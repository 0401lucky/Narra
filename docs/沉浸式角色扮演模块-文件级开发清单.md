## 1. 最推荐的实施总顺序

按这个顺序做，返工最少：

1. 先补 **数据层**（Scenario / Session / Room / Repository）
2. 再补 **导航和页面壳子**
3. 再补 **RoleplayViewModel**
4. 再打通 **底层聊天与上下文装配复用**
5. 最后再补 **RP 输出协议 / narration / dialogue 渲染**

也就是说：

### 正确顺序
- 先有“能打开页面、能选背景图、能绑定角色”的空壳
- 再让它能发消息
- 再让它长得像视觉小说
- 最后再让模型输出更结构化

### 不推荐顺序
- 一开始就做复杂协议
- 一开始就做表情切换
- 一开始就做分支选项
- 一开始就重构全部聊天底层

## 3. 第二阶段：Repository 层

## 3.1 新增：`data/repository/roleplay/RoleplayRepository.kt`
**文件路径**
- `app/src/main/java/com/example/myapplication/data/repository/roleplay/RoleplayRepository.kt`

**职责**
- 场景 CRUD
- Session CRUD
- 启动场景时创建/复用底层 conversation
- 将 Room entity ↔ domain model 转换

**建议接口**
```kotlin
interface RoleplayRepository {
    fun observeScenarios(): Flow<List<RoleplayScenario>>
    suspend fun listScenarios(): List<RoleplayScenario>
    suspend fun getScenario(id: String): RoleplayScenario?
    suspend fun upsertScenario(scenario: RoleplayScenario)
    suspend fun deleteScenario(id: String)

    suspend fun startScenario(
        scenarioId: String,
        conversationRepository: ConversationRepository,
        selectedModel: String,
    ): RoleplaySession

    suspend fun getSessionByScenario(scenarioId: String): RoleplaySession?
    suspend fun getSession(sessionId: String): RoleplaySession?
    suspend fun upsertSession(session: RoleplaySession)
}
```

**实现建议**
- `startScenario(...)`：
  - 如果已有 session，优先返回
  - 如果没有 session，就创建一个新的 `Conversation`
  - 新建 `RoleplaySession`
  - 让 `conversation.assistantId = scenario.assistantId`

**注意**
- 这里不要直接调用 `AiRepository`
- Repository 只负责场景与 session 生命周期，不负责发消息

## 8.2 复用 `PromptContextAssembler`
**不要再做一套 RP 专属记忆/世界书装配器**

### 正确做法
RoleplayViewModel 发送消息时：
1. 取当前 scenario 绑定的 assistant
2. 调 `promptContextAssembler.assemble(...)`
3. 在其 `systemPrompt` 后追加 `RoleplayPromptDecorator` 的 RP 协议提示
4. 再调用 `AiRepository.sendMessageStream(...)`

---

## 8.3 复用 `ConversationRepository`
**不要再做一套 RP 消息表**

### 正确做法
- 一个 RP session 绑定一个 `conversationId`
- 消息仍保存到 `messages` 表
- RP 页面只是在 UI 层做特殊渲染

## 10. 最终“最小可运行版本”定义

只要以下 8 条满足，就算 RP MVP 完成：

1. 有 RP 入口
2. 能创建 RP 场景
3. 场景能绑定一个已有 Assistant
4. 能设置背景图
5. 能设置用户图与角色图
6. 能启动一个 RP session
7. 能在 RP 页面发消息并收到 AI 回复
8. AI 回复能以“视觉小说式界面”显示
