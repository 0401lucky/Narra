# Narra 发布与更新约定

## 1. 渠道

- 正式渠道：`release`
- 开发渠道：`dev`
- 基线渠道：`baseline`

对应包名：

- `release` -> `com.narra.app`
- `dev` -> `com.narra.app.dev`
- `baseline` -> `com.narra.app.baseline`

---

## 2. 版本比较

客户端后续检查更新时：

- 只使用 `versionCode` 判断新旧
- `versionName` 只负责展示

当前规则：

```text
versionCode = major * 10000 + minor * 100 + patch
```

---

## 3. APK 文件名

统一输出格式：

```text
Narra-v{versionName}-{versionCode}-{channel}.apk
```

示例：

- `Narra-v1.0.0-10000-release.apk`
- `Narra-v1.0.0-dev-10000-dev.apk`
- `Narra-v1.0.0-baseline-10000-baseline.apk`
- 当前开发分发包示例：`Narra-v1.1.6-dev-10106-dev.apk`

---

## 4. 服务端更新元数据

建议按渠道分别维护：

- `release.json`
- `dev.json`
- `baseline.json`

推荐字段：

```json
{
  "app_id": "com.narra.app",
  "channel": "release",
  "latest_version_name": "1.1.0",
  "latest_version_code": 10100,
  "minimum_supported_version_code": 10000,
  "apk_url": "https://example.com/narra/Narra-v1.1.0-10100-release.apk",
  "apk_sha256": "建议提供",
  "published_at": "2026-03-22T12:00:00+08:00",
  "release_notes": [
    "新增功能 A",
    "修复问题 B"
  ]
}
```

说明：

- 当前客户端统一按“可选更新”处理，不启用强制更新。
- `minimum_supported_version_code` 仅作为兼容保留字段，建议填写 `0` 或不高于当前已发布版本。

---

## 5. 后续实现建议

当真正开始做应用内更新时，优先补这几项：

1. 客户端版本元数据模型
2. 更新接口拉取与本地缓存
3. SHA-256 校验
4. 下载完成后的安装跳转
5. 失败重试与渠道隔离
