## [1.0.3] - 2026-03-23

### Added

- 助手记忆页新增“共享全局长期记忆”开关
- 助手长期记忆支持直接编辑
- 沉浸式扮演上下文新增“角色长期记忆 / 当前剧情约束”双块注入

### Changed

- 沉浸式扮演默认引入更高采样参数，并在不兼容时自动回退
- AI 帮写会参考最近玩家口吻，建议改为 `plot / info / emotion` 三轴输出
- 长期记忆提取后会先做合并压缩，再写入长期记忆池
- 启用共享长期记忆时，聊天与沉浸式扮演都会优先读写全局长期记忆

### Fixed

- 缓解沉浸式扮演回复起手句、动作模板和情绪标签重复导致的“木感”
- 修正 RP 记忆注入语义不清的问题，避免长期设定与当前剧情状态混在一起
- 修正共享长期记忆开启后仍混入助手私有长期记忆的问题

### Refactored

- 重构 RP 长期记忆与剧情状态记忆的整理、压缩与注入链路
- 重构 RP prompt 装饰逻辑，新增“本轮导演提示”约束
- 统一聊天与 RP 长期记忆的压缩存储策略

### Docs

- 更新当前正式版本到 `1.0.3`

## [1.0.2] - 2026-03-22

### Added

- 沉浸式扮演支持“重回此回合”
- 沉浸式扮演消息支持长按复制
- 沉浸式扮演输入区接入“特殊玩法 -> 转账”

### Changed

- AI 帮写建议区改为内部滚动展示，长文本不再被压缩截断
- AI 帮写点选建议后会自动写入输入框并收起建议区
- 沉浸式扮演与聊天页复用统一的转账卡片展示逻辑

### Fixed

- 统一 `400` / `429` AI 请求错误提示，补充兼容性与限流说明
- 修正 Gemini OpenAI 兼容地址规范化逻辑
- 修复 Tavern PNG 角色卡导入解析，恢复图片角色卡预览与导入

### Refactored

- 沉浸式扮演消息映射补齐特殊玩法与消息状态信息
- 抽出共享转账卡片组件与转账状态更新逻辑

### Docs

- 更新当前正式版本到 `1.0.2`

## [1.0.0] - 2026-03-22

### Added

- 确立正式英文名 `Narra`
- 固定正式 `applicationId` 为 `com.narra.app`
- 建立版本规则：`major.minor.patch`
- 建立渠道规则：`release` / `dev` / `baseline`
- 新增沉浸式角色扮演模块的 `AI帮写 / 剧情建议` MVP

### Changed

- 统一构建产物命名规则为 `Narra-v{versionName}-{versionCode}-{channel}.apk`
- 统一调试包显示名为 `Narra Dev`
- 统一基线包显示名为 `Narra Baseline`

### Docs

- 新增版本策略文档
- 新增发布与更新约定文档
