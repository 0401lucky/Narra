# Changelog

本文件记录 `Narra` 的版本变更。

维护规则：

- 未发布的改动，先写入 `Unreleased`
- 准备发版时，将对应内容移动到正式版本号下
- 每次版本记录尽量按类型归档：
  - `Added`：新增功能
  - `Changed`：已有功能调整
  - `Fixed`：缺陷修复
  - `Refactored`：重构或内部整理
  - `Docs`：文档与发布说明调整

## [1.7.2-dev] - 2026-05-04

### Added

- MiMo 预置音色选择弹窗新增官方预置音色试听，已生成试听音频会复用缓存，避免重复请求。

### Changed

- 普通聊天和沉浸式聊天流式输出改为分帧批量刷新，降低高频重组导致的卡顿。
- 普通聊天页新增滚动轻量渲染模式、键盘弹起跟随、顶部/底部跳转按钮与输出完成后的轻微整理提示。
- 消息操作栏收敛为高频动作，翻译、记忆、预览、导出和分享等低频动作统一放入更多菜单。
- 长状态卡和长 HTML 内容默认折叠，并显示项数、段落数和字数摘要，减少首屏渲染压力。

### Fixed

- 修复角色卡、世界书、记忆和预设插槽中包含 `${...}` 文本时被 Kotlin 正则替换误判为捕获组而生成失败的问题。
- 状态块解析增加异常兜底，角色卡输出异常格式时不再直接中断消息显示。

### Docs

- `dev` 渠道构建版本推进到 `1.7.2-dev (10702)`，继续通过 Cloudflare R2 分发 APK。

## [1.7.0-dev] - 2026-05-03

### Added

- 线上 `voice_message` 接入小米 MiMo TTS，生成成功后可播放本地真实音频，并继续保留原文字兜底
- 新增独立 MiMo 语音合成设置，支持安全保存 API Key、全局默认音色、角色级预置音色与文本设计音色
- MiMo 语音合成支持可配置 Base URL，可使用 Token Plan 专属入口、普通官方入口或第三方 OpenAI 兼容中转
- 角色文本设计音色支持从角色人设自动生成 VoiceDesign 描述，并允许继续手动调整
- 角色音色支持 MiMo VoiceClone，可导入 `wav/mp3` 声音样本并固定给单个角色使用
- 新增 MiMo 请求客户端、本地音频保存与后台语音生成协调器，失败仅更新语音卡片状态，不影响整条助手消息

### Changed

- 语音卡片从假播放切换为 Android `MediaPlayer` 播放本地音频，并展示生成中、可播放、失败等状态
- MiMo 设置页新增模型选择、Key 显示/隐藏、Base URL 编辑与连通性测试，便于定位 Key 或接口地址问题
- 修复角色聊天消息高度变化时列表位移动画可能造成卡片叠压的问题
- 信箱主动来信、信箱回信、沉浸式聊天和 AI 帮写生成中新增更明确的停止入口
- 线上沉浸式聊天进入页面时不再因加载旧历史触发久未联系自动补话
- 沉浸式背景图降低解码尺寸并移除双图层切换，减少重渲染导致的白屏或崩溃风险
- `voice_message` 的 `actionMetadata` 增加音频状态、路径、MIME、模型和音色信息，无需数据库迁移

### Docs

- `dev` 渠道构建版本推进到 `1.7.0-dev (10700)`

## [1.6.6-dev] - 2026-05-02

### Added

- 新增聊天记录分段压缩，旧聊天会按段生成摘要档案，模型请求改为使用“总摘要 + 近期分段 + 最近原文窗口”

- 线上手机头像新增双向“拍一拍”，支持用户拍角色头像，也支持角色拍用户头像

### Changed

- 角色扮演摘要模板升级为剧情档案格式，保留剧情进展、当前状态、关系变化、角色状态、伏笔和近期触发点

- 侧栏入口从“手动总结记忆”调整为“整理摘要与记忆”，明确同一入口会分别执行聊天摘要和长期记忆提炼

### Fixed

- 修复线上手机引用发送时没有把引用内容落到用户消息快照里的问题，群聊引用也会保留被引用角色名和预览
- 修复 AI 帮写入口直接把第一条建议塞进输入框的问题，现在会恢复展示三条可选建议
- 修复头像拍一拍触发的发送任务没有接入取消发送句柄的问题

### Refactored

- Room 升级到 schema 38，新增 `conversation_summary_segments` 分段摘要表，原始聊天记录继续完整保留

### Docs

- `dev` 渠道构建版本推进到 `1.6.6-dev (10606)`，APK 上传 Cloudflare R2 后由 `docs/updates/dev.json` 分发
