# Changelog

本文件记录 `Narra` 的版本变更。

维护规则：

- 未发布的改动，先写入 `Unreleased`
- 准备发版时，将对应内容移动到正式版本号下
- 每次版本记录尽量按类型归档：
  - `Added`：新增功能
  - `Changed`：已有功能调整
  - `Fixed`：缺陷修复
  - `Refactored`：重构或内部整理
  - `Docs`：文档与发布说明调整

## [1.1.6] - 2026-04-09

### Added

- 特殊玩法新增 `惩罚` 卡，支持按 `方式 / 次数 / 强度 / 原因 / 附注` 发送给角色，聊天页与 RP 页统一可用

### Changed

- “我的手机”语义正式收口为“用户内容 + 触发点”，仍服务角色反应，但内容主体固定来自用户本人
- `惩罚` 卡沿用现有特殊玩法协议与卡片系统，不额外引入状态机，发送后由角色在正文中直接承接反应

### Fixed

- 修复 `线上模式` 顶部栏与系统状态栏重叠，刘海屏下标题和按钮不再被压住
- 修复 `线上模式` 用户消息气泡宽度固定的问题，短句改为按内容收缩、长句仍保留上限
- 修复旧版“我的手机”缓存继续展示错误语义的问题，进入页面会自动判旧并提示重新生成

### Refactored

- 手机快照增加语义版本字段，并支持按 `conversationId + ownerType` 定向删除，清理“我的手机”旧缓存时不会误删“TA 的手机”
- 特殊玩法协议扩展支持 `type="punish"`，继续复用同一条 `SPECIAL` 消息结构、解析器和 UI 卡片骨架

### Docs

- `dev` 渠道应用内更新推进到 `1.1.6-dev (10106)`，同步更新下载地址、校验值、发版说明与版本记录

## [1.1.5] - 2026-04-09

### Added

- 查手机玩法扩展为双对象快照，同一会话下可分别查看 `TA的手机` 与 `我的手机`
- 角色查看用户手机后会写入会话级观察状态，并立即在主聊天回写一条可见事件提示
- 沉浸式新增 `线上模式`，提供手机聊天壳、时间分隔、查手机入口与正常强度旁白
- 线上模式支持长按消息引用，发送后显示引用预览条，并支持点击回跳到原消息
- 线上模式新增最小事件流：截图聊天、撤回最近一条消息、对方连发提示、正在输入提示
- 沉浸式进入线上模式后支持按真实时间差触发一次“久未联系补偿开场”

### Changed

- RP 运行模式正式收口为 `OFFLINE_LONGFORM / OFFLINE_DIALOGUE / ONLINE_PHONE`，场景编辑页与沉浸设置页都可立即切换
- 手机快照生成上下文增加 `ownerType / viewerName / timeGapContext`，让“谁在看谁的手机”和久未联系状态都能进入生成链路
- 线上模式提示词允许角色主动引用最近消息，并允许基于“看过用户手机”的既有事件继续互动
- 查手机生成链路继续保留聊天建议模型，但请求按板块顺序拆分生成，降低一次性刷新超时概率

### Fixed

- 修复 Tavern / 角色设定中的 `{{User}} / {{char}}` 等占位符未替换，导致模型把原始模板字符直接说出来的问题
- 修复查手机页进入与生成时误读空设置导致“当前未配置可用提供商”以及标题回退成默认助手的问题
- 修复查用户手机在全量生成成功时没有写入观察状态，导致后续聊天继承失效的问题
- 修复线上模式中引用消息上下文未传给模型，导致角色无法真正“接住”被引用内容的问题

### Refactored

- 消息模型、Room 与 RP UI 映射补入 `replyTo* / isRecalled / systemEventKind` 字段，为线上生态功能铺底
- 新增 `phone_observations` 与 `roleplay_online_meta` 数据层，统一承载手机查看后效和线上模式去重状态
- 手机查看状态注入策略改为“未反馈默认注入，反馈后仅在相关话题再次注入”，避免角色每轮机械重复

### Docs

- `dev` 渠道应用内更新推进到 `1.1.5-dev (10105)`，同步更新下载地址、校验值、发版说明与版本记录

## [1.1.4] - 2026-04-08

### Added

- 沉浸式长文模式新增智能分段兜底，在模型没有主动保留自然空行时也能拆成更自然的小说段落
- 搜索结果协议升级为 `answer + items[] + stable id`，并新增聊天内搜索选择器与搜索结果预览闭环

### Changed

- RP 提示词强化了角色锁定、互动边界和剧情推进规则，并把助手描述与创作者导演说明正式纳入 RP 提示词链路
- 聊天页搜索入口升级为搜索选择器，支持直接查看当前搜索源、LLM 代理状态和默认返回条数

### Refactored

- 长文解析与沉浸式/阅读模式展示统一复用同一套段落拆分逻辑，避免不同页面各自按换行硬切
- 搜索结构化解析与搜索工具输出统一复用同一套 `query / answer / items[]` 协议，为后续正文内引用闭环继续铺路

### Docs

- `dev` 渠道应用内更新已推进到 `1.1.4-dev (10104)`，并同步更新下载地址、校验值与版本说明

### Added

- 聊天页思考内容升级为步骤式时间线卡片，支持多段思考步骤落库与流式展示
- 聊天页消息新增统一“更多操作”面板，支持消息级选中复制、渲染预览、Markdown 导出和系统分享
- 聊天页新增消息级 WebView 工作台，可渲染本地 HTML 预览并以内置浏览模式打开引用链接
- 沉浸式长文模式新增智能分段兜底，在模型没有主动保留自然空行时也能拆成更自然的小说段落
- 搜索结果协议升级为 `answer + items[] + stable id`，并新增聊天内搜索选择器与搜索结果预览闭环
- Tavern PNG / JSON 角色卡导入支持深层 `character_book` 世界书解析
- 角色卡导入时会同步落库助手专属世界书，并自动挂回助手扩展配置
- RP 世界书匹配支持 Tavern 常见的逗号分隔触发词
- RP 顶部状态新增更可靠的世界书命中反馈与 Prompt 调试信息
- 沉浸式页面新增独立“沉浸设置”页与阅读模式入口
- 世界书支持独立 lorebook / 世界书 JSON 导入
- 聊天页与沉浸式输入框新增“展开编辑”入口，可拉起大尺寸草稿编辑面板，便于浏览和修改长输入
- 沉浸式消息区新增快捷滚动按钮，支持按用户上滑 / 下滑方向显示一键回到顶部 / 回到底部入口
- “当前角色已改绑”提示框新增“不再显示”选项，勾选并继续旧剧情后可记住偏好，后续不再重复弹出
- 特殊玩法从单一转账扩展为转账、邀约、礼物、委托四类卡片，聊天页与沉浸式扮演页统一支持发送
- 礼物特殊玩法升级为自动生图 V1，礼物卡支持生成中、成功、失败三态并直接展示礼物图

### Changed

- 聊天页思考展示对齐 `RikkaHub` 风格，支持“展开更多步骤 / 收起思考过程”、生成中预览和单步耗时标题
- 聊天页支持从历史用户消息回退并恢复输入区，用户可修改后重新发送后续对话
- RP 提示词强化了角色锁定、互动边界和剧情推进规则，并把助手描述与创作者导演说明正式纳入 RP 提示词链路
- 聊天页搜索入口升级为搜索选择器，支持直接查看当前搜索源、LLM 代理状态和默认返回条数
- AI 帮写改为直接生成草稿并填入输入框，发送前不落库
- 进入沉浸式会话页时会优先定位到最新一条剧情回复，减少长剧情场景下反复手动下滑的成本
- RP 纯文本回复会自动拆分为旁白/动作与对白两种展示
- 多处设置页、聊天页与 RP 页面视觉样式做了统一优化
- 沉浸式页面支持默认长剧情字数、身份条显示和状态条显示等全局偏好
- 沉浸式页面、阅读模式与设置页统一了背景直出与半透明沉浸视觉
- RP 长文本对白现在支持引号内对白高亮
- Provider 配置页支持按协议区分 OpenAI Compatible / Anthropic，并为 OpenAI Compatible 增加 Chat Completions / Responses API 模式切换
- 自定义 OpenAI Compatible Provider 现在支持配置 `chatCompletionsPath`，用于适配第三方兼容端点的非标准路径
- 聊天、翻译与提示词增强链路已开始按 Provider 文本接口模式选择 `chat/completions` 或 `responses`
- 沉浸设置页切换模型后会优先更新本地草稿，降低“视觉切换成功但实际发送仍用旧模型”的问题
- 功能模型支持“跟随默认 / 单独指定 / 关闭”三态，可显式关闭聊天建议、翻译、记忆、搜索、礼物生图等能力
- 聊天页底部输入栏进一步收紧，整体更贴底、更紧凑
- 聊天页历史消息滚动体验优化，减少滚动过程中的重渲染与闪动感

### Fixed

- 修复 Tavern 角色卡自带世界书导入后未真正进入数据库的问题
- 修复角色专属世界书可能在其他助手页面串号的问题
- 修复 RP 页面世界书命中状态与调试 Dump 不一致的问题
- 修复助手扩展页只能看到已关联世界书，无法把任意世界书挂到任意助手的问题
- 修复 Android Studio 旧运行配置残留导致的启动类定位错误
- 修复沉浸式阅读模式与设置页近期引入的编译阻塞
- 修复沉浸式“对白模式”在真机上的文本排版可读性问题，为旁白与角色对白补回半透明承载层，并让未带引号的角色对白也能按对白样式高亮显示
- 修复沉浸式对白模式里内嵌引号短词被错误拆成独立对白块，导致上下文断裂、出现“上文不接下文”的排版问题
- 修复 RP 协议输出中“未包 `<narration>` 标签的叙述段”被误判成角色对白，导致旁白未分段且文字颜色掉成对白色的问题
- 修复第三方 RP 协议输出把动作/叙述混写进 `<dialogue>` 标签时，旁白未拆分出来的问题，降低对白块内旁白串色和串段概率
- 修复聊天页在浏览历史消息时滚动不连贯、停下后出现明显闪动的问题

### Refactored

- 聊天消息模型、流式事件和 Room 持久化新增 reasoning steps 结构，为后续多段思考和工具时间线打底
- 会话级导出/分享能力已拆出消息级格式化函数，聊天消息的预览、分享和导出复用同一套消息快照构建逻辑
- 长文解析与沉浸式/阅读模式展示统一复用同一套段落拆分逻辑，避免不同页面各自按换行硬切
- 搜索结构化解析与搜索工具输出统一复用同一套 `query / answer / items[]` 协议，为后续正文内引用闭环继续铺路
- 重构资料导入链路，区分 Tavern 角色卡与通用上下文数据包导入逻辑
- 重构沉浸式消息映射与长文本消息渲染逻辑
- 大幅拆分聊天页、Provider 详情页、消息气泡、沉浸设置页、沉浸主页面、沉浸对话面板与沉浸式 ViewModel 的大文件，逐步转为编排层 + support 层结构
- 抽离 Roleplay 的观察、建议、场景操作、发送执行与上下文更新 support，降低沉浸式主逻辑文件复杂度
- 聊天消息渲染链路增加稳定缓存与轻量渲染分支，降低长文本与富文本在历史浏览时的测量压力

### Docs

- 约定当前开发分发包版本推进到 `1.1.0-dev`
- 新增《项目收尾清单》与《人工回归测试清单》，把 Responses API、自定义端点与主链路回归验证纳入正式收尾计划
- `dev` 渠道应用内更新下载源已切到 Cloudflare R2 自定义域名 `download.lsa1230.dpdns.org`

## [1.0.3] - 2026-03-23

### Added

- 助手记忆页新增“共享全局长期记忆”开关
- 助手长期记忆支持直接编辑
- 沉浸式扮演上下文新增“角色长期记忆 / 当前剧情约束”双块注入

### Changed

- 沉浸式扮演默认引入更高采样参数，并在不兼容时自动回退
- AI 帮写会参考最近玩家口吻，建议改为 `plot / info / emotion` 三轴输出
- 长期记忆提取后会先做合并压缩，再写入长期记忆池
- 启用共享长期记忆时，聊天与沉浸式扮演都会优先读写全局长期记忆

### Fixed

- 缓解沉浸式扮演回复起手句、动作模板和情绪标签重复导致的“木感”
- 修正 RP 记忆注入语义不清的问题，避免长期设定与当前剧情状态混在一起
- 修正共享长期记忆开启后仍混入助手私有长期记忆的问题

### Refactored

- 重构 RP 长期记忆与剧情状态记忆的整理、压缩与注入链路
- 重构 RP prompt 装饰逻辑，新增“本轮导演提示”约束
- 统一聊天与 RP 长期记忆的压缩存储策略

### Docs

- 更新当前正式版本到 `1.0.3`
