# 版本功能添加 / 优化记录

## 1.7.2-dev

### 新增功能
- MiMo 预置音色选择弹窗新增官方预置音色试听，已生成试听音频会复用本地缓存，避免再次打开时重复调用接口。

### 体验优化
- 普通聊天与沉浸式聊天流式输出改为分帧批量刷新，降低高频重组带来的卡顿和掉帧。
- 普通聊天页新增滚动轻量渲染模式，滚动或流式输出期间优先展示轻量文本，停稳后再恢复富文本。
- 流式输出完成后延迟短暂重排，并显示“正在整理格式…”提示，避免完成瞬间突然卡顿却没有反馈。
- 普通聊天消息列表新增顶部/底部跳转按钮，长对话中快速回到开头或最新消息更方便。
- 键盘弹起时仅在贴近底部或自动跟随状态下轻推列表，输入时最新消息不易被键盘遮住，也不会打断翻历史消息。
- 消息操作栏收敛为复制、必要重试、更多操作；翻译、记忆、预览、导出和分享等低频动作放入更多菜单。
- 长状态卡和长 HTML 内容默认折叠，并显示项数、段落数和字数摘要，减少首屏渲染压力。

### 缺陷修复
- 修复角色卡、世界书、记忆和预设插槽中包含 `${...}` 文本时被 Kotlin 正则替换误判为捕获组而导致生成失败的问题。
- 状态块解析增加异常兜底，角色卡输出异常格式时不再直接中断消息显示。
- 更多操作菜单支持滚动，小屏幕或多附件消息下不会因为操作项过多而被底部裁掉。

### 技术优化
- 新增通用 `StreamingReplyBuffer` 流式缓冲批处理，普通聊天与角色扮演共用分帧发布逻辑。
- 富文本、推理时间线、状态卡和 HTML 卡片在流式与滚动阶段减少昂贵解析，停稳后再恢复完整渲染。

## 1.7.0-dev

### 新增功能
- 线上 `voice_message` 接入小米 MiMo TTS：助手语音消息完成后后台生成真实音频，生成成功即可播放本地 `wav` 文件。
- 新增独立“MiMo 语音合成”设置页，默认关闭，填写 API Key 并开启后生效，不并入现有聊天模型供应商。
- MiMo 语音合成支持单独配置 Base URL，可接入 Token Plan 专属入口、普通 MiMo 官方入口或第三方 OpenAI 兼容中转。
- 支持全局默认音色与角色级音色配置：角色可继承全局、选择预置音色、填写 VoiceDesign 文本设计音色，或单独关闭语音生成。
- 角色文本设计音色新增“从人设生成”，可根据角色名称、简介、系统提示、场景、开场白和标签自动生成可编辑的 VoiceDesign 描述。
- 角色音色新增 MiMo VoiceClone 声音克隆：可导入本地 `wav/mp3` 样本并固定给某个角色使用。
- 支持 MiMo 预置音色 `mimo_default`、冰糖、茉莉、苏打、白桦、Mia、Chloe、Milo、Dean。

### 体验优化
- 语音生成不阻塞文本回复落地；未配置 Key、关闭开关、网络异常、响应异常或音频保存失败时，仍完整显示原始文字。
- 语音卡片替换原假播放逻辑，按状态显示“生成中 / 可播放 / 生成失败”，失败时保留展开与复制文字能力。
- 语音设置页新增 Key 显示/隐藏、模型选择与连通性测试，方便区分 Key、Base URL、模型参数问题。
- 修复角色聊天消息列表高度变化时可能被位移动画挤压覆盖的问题，避免语音、心声和对白卡片互相叠住。
- 信箱主动来信、信箱回信和沉浸式聊天生成中新增明确“停止”入口，取消后不再让模型请求继续占用当前回合。
- 沉浸式 AI 帮写生成中可直接停止，不再只显示不可操作的“生成中”状态。
- 线上模式新增场景级“主动回复”开关，默认关闭；只有开启后才会在久未联系时补开场，避免刚进入页面就自动发消息。
- 沉浸式背景图加载限制最大解码边，移除背景切换时双图层 Crossfade，降低返回主页面时白屏或崩溃的内存压力。
- 多条语音消息分别保存独立音频文件，播放状态互不覆盖。

### 技术优化
- 新增 `MimoTtsClient`，通过 `api-key` 与 `Authorization: Bearer` 请求头调用 OpenAI 兼容 chat completions 端点，解析 `choices[0].message.audio.data`。
- Base URL 可填写 `/v1` 或完整 `/chat/completions`，客户端会自动解析最终请求地址，避免重复拼接。
- VoiceClone 样本复制到应用私有目录保存，导入时校验 `wav/mp3` 与 MiMo Base64 10 MB 上限；合成时以 `data:{mime};base64,...` 传入 `audio.voice`。
- 新增 `AudioFileStorage`，将 Base64 音频保存到应用私有目录 `generated_voice_messages`，并校验非法 Base64。
- 新增 `VoiceSynthesisCoordinator`，只处理助手已完成消息中的 `voice_message`，成功/失败都只回写该语音 part 的 metadata。
- `ChatMessagePart.actionMetadata` 新增 `audio_status`、`audio_path`、`audio_mime_type`、`audio_error`、`tts_model`、`tts_voice_mode`、音色 ID 或音色 prompt hash，无需 Room 迁移。
- MiMo API Key 复用现有安全存储机制保存，DataStore 中只保留脱敏后的语音合成配置。

- 修复群聊引用角色消息后，发送出去的用户消息缺少被引用角色名和引用预览的问题。
- 修复头像拍一拍触发的发送任务没有接入取消发送句柄的问题。

- 摘要生成链路支持只压缩最近窗口之外且尚未覆盖的旧消息，超长历史会分批生成多个分段摘要。
