# 版本功能添加 / 优化记录

## 1.1.7

### 新增功能
- 沉浸设置新增 `线上模式显示旁白` 开关，想体验纯线上聊天感的用户可以直接关闭独立旁白展示。

### 体验优化
- 线上模式关闭旁白开关后，会尽量只保留对白气泡和系统事件提示，整体更接近纯手机聊天界面。

### 技术优化
- 线上模式旁白开关同时接入了 prompt 约束和 UI 过滤，关闭后不只是“隐藏显示”，也会尽量减少模型继续输出独立旁白段。

## 1.1.6

### 新增功能
- `线上模式` 的查手机继续沿用双对象入口，但“我的手机”语义正式收口为“用户内容 + 触发点”：手机主人固定是用户本人，内容用于给角色后续反应提供线索。
- “我的手机”快照新增语义版本控制，旧版缓存会在进入页面时自动判旧失效，并提示重新生成，避免继续看到错位内容。
- 特殊玩法新增 `惩罚` 卡，用户可以把“抽打/教训”类互动整理成结构化卡片发送，角色会在正文里直接承接反应。

### 体验优化
- 修复 `线上模式` 顶部栏与系统状态栏重叠的问题，刘海屏和沉浸式状态栏下不再压住标题与按钮。
- `线上模式` 的用户消息气泡改为短句自适应、长句封顶，更接近真实聊天应用的消息宽度表现。
- “我的手机”在消息、搜索、备忘录、相册、购物等板块里都会明确站在用户侧生成，但会优先保留最能触发角色反应的线索。
- `惩罚` 卡支持方式、次数、强度、原因和附注，聊天页与 RP 页都能直接编辑、预览并发送。

### 技术优化
- `PhoneSnapshot` 增加 `contentSemanticsVersion`，用于区分用户手机内容的旧语义与新语义。
- `PhoneSnapshotDao / PhoneSnapshotRepository` 新增按 `conversationId + ownerType` 定向删除快照的能力，清理“我的手机”旧缓存时不会误删“TA 的手机”。
- 查手机 prompt 按 `ownerType` 分支生成，补充 `messages[].is_owner` 与用户侧联系人网络约束，降低模型把用户手机写成角色手机的概率。
- 线上模式消息项新增独立气泡布局模式，只对用户普通对白启用自适应宽度，不影响角色消息和特殊玩法卡片。
- 特殊玩法协议扩展为支持 `type="punish"`，继续沿用同一条 `SPECIAL` 消息链路和 XML 解析器，不新增额外状态流。

## 1.1.5

### 新增功能
- 查手机扩展为双对象查看：同一会话下既能查看角色手机，也能查看用户手机。
- 角色查用户手机后，会把关键发现写成会话级观察状态，并立即在主聊天中插入一条可见事件提示。
- 沉浸式新增 `线上模式`：提供手机聊天壳、气泡消息流、时间分隔线、查手机入口与 AI 帮写共存体验。
- 线上模式支持长按消息引用，发送后会显示引用预览条，并支持点击回跳到原消息。
- 线上模式新增基础事件流：截图聊天、撤回最近一条消息、对方连发提示、正在输入提示。
- 进入线上模式后，会按真实时间差触发一次“久未联系补偿开场”，让角色能更自然表现等待感与后效。

### 体验优化
- 旁白在 `线上模式` 中保持正常存在感，不会被压缩成只剩少量系统提示。
- `查手机` 的上下文会明确区分“手机主人 / 查看者 / 关系方向”，让用户手机与角色手机的内容风格真正分开。
- 角色在主聊天和线上模式里都能延续“看过用户手机”的后效，用户追问时也能优先得到针对性反应。

### 技术优化
- 手机快照模型扩展为 `ownerType` 维度，按 `conversationId + ownerType` 分离存储两份快照。
- 新增 `PhoneObservationState` 和 `RoleplayOnlineMeta`，分别用于记录手机查看状态与线上模式时间桶/事件去重信息。
- 消息模型与 Room 增加 `replyToMessageId / replyToPreview / replyToSpeakerName / isRecalled / systemEventKind` 字段，为线上社交生态打底。
- 查手机生成继续沿用聊天建议模型，但改为按板块顺序拆分请求，配合更高超时阈值，降低全量刷新 timeout 概率。

## 1.1.4

### 新增功能
- 沉浸式长文模式新增智能分段兜底：当模型没有主动保留自然空行时，客户端会按句号、问号、引号收束和动作停顿做保守切段，避免长文整坨堆在一起。
- RP 提示词正式补入助手 `description` 与 `creatorNotes`，让角色核心设定和导演说明真正进入沉浸式链路，而不再只停留在资料字段里。
- 搜索结果协议升级为 `query / answer / items[]`，每条搜索结果都会带稳定 `id`，方便后续回答引用和结果回看。
- 聊天页新增搜索选择器面板，可在聊天内直接查看当前搜索源、LLM 搜索代理状态、默认返回条数，并跳转搜索设置页。
- 搜索型消息新增搜索结果预览面板，可直接回看本轮查询词、搜索摘要和命中的结果列表。

### 体验优化
- RP 长文模式强化了“角色锁定 + 流畅推进”规则：要求先回应当前动作/情绪/问题，再以角色临场反应推动关系、信息或局势。
- 长文小说模式强调自然段落感优先于凑字数，每段只承载一个主要动作、情绪重心或推进点，减少一整坨大段落。
- 沉浸式主界面与阅读模式统一复用同一套段落结果，不再出现一个页面有分段、另一个页面还是一整块的割裂感。

### 技术优化
- `RoleplayLongformMarkupParser` 从“按单个换行拆段”升级为“双层策略”：先尊重模型原始空行，再在缺少自然段落时触发智能分段。
- 智能分段只改变段落边界，不改写正文内容，也避免把 `<char>` 或 `<thought>` 标记内部强行切断。
- `RoleplayConversationSupport.buildDynamicDirectorNote` 升级为更具体的导演提示，明确当前关系张力、优先推进点、当前阻碍和上一轮线索承接。
- `SearchModelExecutor` 的 Responses / Anthropic 搜索解析统一收口到 `answer + items[]` 结构，并对引用 URL 做规范化去重。
- `SearchWebTool` 输出同时保留新 `items[]` 和兼容 `results[]`，在不打断旧消息的前提下为新 UI 提供结构化结果。

## 1.1.3

### 新增功能
- 聊天页思考区域升级为步骤式时间线卡片，支持按步骤展示思考标题、耗时与正文。
- 聊天消息模型新增 `reasoningSteps`，流式回复和最终落库都可以保存完整思考步骤，而不再只保留单段字符串。
- Room 消息表新增 `reasoningStepsJson`，历史 `reasoningContent` 会在数据库迁移时自动回填成单步思考记录。
- 聊天页消息新增统一“更多操作”面板，支持选中复制、渲染预览、消息级 Markdown 导出和系统分享。
- 聊天页新增消息级 WebView 工作台，可渲染消息 HTML 预览，并支持在内置浏览模式打开引用链接。

### 体验优化
- 聊天页思考卡片对齐 `RikkaHub` 风格，折叠态默认只展示最后 2 条步骤，并提供“展开更多步骤 / 收起思考过程”入口。
- 生成中思考支持预览态：会保留最近步骤的正文预览，并在卡片内部自动滚到最新内容，减少用户反复手动展开。
- 已完成的思考步骤会根据最后一个独立加粗标题行显示步骤标题；没有标题时，生成中显示耗时，结束后回落为“深度思考”。
- 聊天页现在支持从历史用户消息回退到输入区，自动清空该消息之后的轮次，并把文本与附件一起恢复到待发送状态。

### 技术优化
- 聊天流式事件从单一 `ReasoningDelta` 扩展为 `ReasoningStepStarted / ReasoningStepDelta / ReasoningStepCompleted` 三段式事件。
- `StreamingReplyBuffer` 现在同时维护完整思考步骤和可见思考步骤，逐帧推进时能正确处理步骤开始、增量文本和结束时间。
- `ConversationAssistantRoundTripRunner` 在无特殊更新时直接复用内存中的最终消息列表，减少一次额外读库。
- AI 网关在无法创建流式客户端时会自动回退到非流式请求，避免异常环境下整轮回复直接失败。
- 会话级导出/分享链路已抽出消息级格式化函数，消息预览、导出、分享和选中复制复用同一套消息快照构建逻辑。

## 1.1.2

### 新增功能
- 聊天页与沉浸式扮演页的“特殊玩法”从单一转账扩展为四类：`转账`、`邀约`、`礼物`、`委托`。
- 特殊玩法新增统一选择面板与独立编辑面板，支持在聊天页和 RP 页以同一套交互发送玩法卡片。
- 新增邀约卡、礼物卡、委托卡的结构化展示，后续继续扩玩法时可直接沿用当前骨架。
- 礼物特殊玩法升级为“自动生图 V1”：用户发送礼物卡后会立即进入生成态，后台异步补出礼物图，并直接绑定回原礼物卡。
- 设置页新增独立“礼物生图模型”角色，礼物自动生图不再依赖当前聊天模型是否本身支持生图。

### 体验优化
- 聊天页与 RP 页统一复用特殊玩法卡片渲染逻辑，视觉风格与交互入口保持一致。
- RP 页新增基于最近角色情绪的场景氛围反馈，顶部会展示当前氛围，背景遮罩和角色高亮会随情绪联动变化。
- 特殊玩法文案镜像、复制内容和消息摘要回退文本已补齐，聊天记录、导出文本与后续标题摘要可读性更好。
- 礼物卡新增`生成中 / 成功 / 失败`三态，成功后直接在卡片内部展示缩略图，不会额外插入独立图片消息。
- 礼物卡生成中补了固定比例占位，图片回填后列表不会出现明显跳动；未配置礼物生图模型时，仍会按普通礼物卡正常发送并提示原因。

### 技术优化
- 特殊玩法底层数据结构已从“转账专用”扩展为通用玩法模型，支持后续继续追加更多卡片类型而不必重做整条链路。
- 模型协议由原先只支持转账标签，扩展为通用 `<play />` / `<play-update />` 协议，便于助手主动生成更多互动卡片。
- RP 消息映射中的特殊玩法类型已收口为统一的 `SPECIAL_PLAY` 展示通道，减少后续新增玩法时的 UI 枚举分叉。
- 礼物卡在不新增消息 Part 类型的前提下，扩展了受控元数据字段，用于记录礼物图的状态、图片地址、类型和失败原因。
- 聊天页与 RP 页都已接入礼物图异步生成协调器：发送礼物卡后，助手回复链路与礼物生图链路并行执行，互不阻塞。
- 礼物生图 Prompt 先走隐藏优化，再在失败时回退到模板 Prompt；图片生成链路补了超时兜底，避免礼物卡长期停在处理中。
